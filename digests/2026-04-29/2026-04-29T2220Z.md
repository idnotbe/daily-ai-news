## #34 HERMES.md in commit messages causes requests to route to extra usage billing

- **출처**: Anthropic (GitHub Issues) | **발행**: 2026-04-29 00:00 UTC
- **링크**: https://github.com/anthropics/claude-code/issues/53262
- **키워드**: claude-code, billing-bug, git-integration, api-routing, cost-management

**📌 핵심**
- Claude Code에서 git 커밋 메시지에 대문자 'HERMES.md' 문자열이 포함될 경우, API 요청이 Max 플랜 할당량이 아닌 추가 과금(extra usage) 경로로 잘못 라우팅되는 심각한 빌링 버그가 공개됐다.
- 실제 피해 사례로, 주간 잔여 용량 86%가 남은 월 $200 Max 플랜 사용자가 예상치 못한 추가 요금으로 $200.98을 청구받았다.
- 버그의 원인은 Claude Code가 최근 git 커밋 내역을 시스템 프롬프트에 포함시키고, 서버측 라우팅 로직이 'HERMES.md' 문자열을 특수 트리거로 인식해 과금 방식을 변경하기 때문이다.
- 'HERMES.md'라는 파일 자체가 존재하지 않아도 커밋 메시지 내용만으로 트리거되며, 소문자 'hermes.md'는 해당 없다.

**💡 컨설턴트 관점**
- Claude Code를 도입한 고객사 중 Max 플랜 사용자라면 이번 달 청구서를 즉시 확인하도록 안내하고, git 커밋 히스토리에 'HERMES.md' 문자열 포함 여부를 점검(`git log --all --grep='HERMES.md'`)하는 간단한 조치를 제안할 수 있다.
- 이 사례는 AI 코딩 도구가 외부 컨텍스트(git 히스토리 등)를 시스템 프롬프트에 자동 포함하는 구조적 리스크를 보여주므로, 고객사의 AI 툴 도입 시 과금 모니터링 체계와 비용 이상 감지 알림 설정을 표준 운영 절차에 포함시킬 것을 권고해야 한다.
